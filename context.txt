1. PROJECT OVERVIEW
================================================================================

TITLE:
  Explainable, Multi-Agent AI-Based Adversarial Threat Detection &
  Decision Framework for Facial Biometric Systems

CORE INNOVATION (3 novelties):
  1. Dynamic Threshold Engine — replaces static/adaptive thresholds using
     4-factor context-aware scoring (confidence, anomaly, embedding drift,
     image quality). Addresses the "one-size-fits-all" limitation from
     Park et al. (2024) Risk Analysis paper.
  2. Advocate–Judge Multi-Agent Debate — LLM-backed Proponent & Opponent
     agents argue, a Judge agent validates. Implements LLM-as-a-Judge
     paradigm (prevents hallucination propagation from single-agent systems).
  3. Grad-CAM / XAI layer — shows WHERE on the face the adversarial
     perturbation was detected.

PIPELINE (end-to-end):
  Input Image
    → MTCNN Face Alignment & Cropping (removes background/shoulder noise)
    → Preprocessing (OpenCV, PIL)
    → Feature Extraction (ResNet-18 CNN)
    → Adversarial Detection Model → label: NORMAL / ADVERSARIAL / SUSPICIOUS
    → Dynamic Threshold Engine → context-aware threshold & risk level
    → Advocate Agent (Proponent) argues image is LEGITIMATE
    → Advocate Agent (Opponent) argues image is ADVERSARIAL
    → Judge Agent validates/overrides the debate
    → Grad-CAM XAI heatmap
    → Final Decision Report (JSON + heatmap)

DATASET:
  1. CASIA-WebFace — 10,000 identity subset (clean base features)
  2. LFW (Labeled Faces in the Wild) — 10,000 images subset
  
  The datasets are strictly balanced (1:1 ratio) to ensure the anomaly head learns 
  to accept "in-the-wild" consumer-grade selfies without false-positives.

TRAINING APPROACH (2-phase):
  Phase 1 — train ResNet-18 on CLEAN faces only (binary: 0=clean)
             Learn face features before introducing adversarial examples.
  Phase 2 — generate adversarial images (FGSM + PGD) from trained Phase 1
             model, then fine-tune as 3-class detector (normal/adversarial/
             suspicious).

agents/llm_factory.py (NEW — the key file):
  - Reads LLM_PROVIDER from .env
  - Returns ChatGroq or ChatOpenAI instance
  - Called by advocate.py and judge.py via get_llm()
  - Model resolution order: explicit arg > LLM_MODEL env > provider default
  - Provider defaults: groq → llama3-70b-8192 | openai → gpt-4o

agents/advocate.py:
  - AdvocateAgent(role="proponent"|"opponent") uses get_llm() internally
  - run_advocates(cnn_str, threshold_str) → (pro_out, opp_out)
  - format_debate_for_judge(pro, opp) → debate transcript string

agents/judge.py:
  - JudgeAgent() uses get_llm() with temperature=0.1 (more deterministic)
  - verdict(debate_transcript, threshold_str) → JudgeVerdict dataclass
  - JudgeVerdict fields: final_decision, risk_level, reasoning, confidence,
                         override (bool), override_reason

pipeline.py (the orchestrator):
  Steps: preprocess → CNN → threshold → advocates → judge → gradcam → JSON
  CLI: python pipeline.py --image <path> --checkpoint <path>
  Output: results/<stem>_report.json + results/<stem>_gradcam.png

6. KEY TECHNICAL DECISIONS & ARCHITECTURE NOTES
================================================================================

CNN Model (model/cnn_model.py):
  - Backbone: ResNet-18 (pretrained on ImageNet)
  - 3 output heads:
      1. Classifier head → 3 logits (normal/adversarial/suspicious)
      2. Embedding head  → 512-d L2-normalized face embedding
      3. Anomaly head    → scalar [0,1] anomaly score
  - 3 classes: Config.CLASS_NAMES = ["normal", "adversarial", "suspicious"]

Dynamic Threshold Engine (model/threshold.py):
  - Formula: T = 0.35*(1-conf) + 0.30*anomaly + 0.20*drift + 0.15*(1-quality)
  - T < 0.25  → trust CNN as-is (LOW risk)
  - T 0.25-0.50 → upgrade "normal" → "suspicious" (MEDIUM risk)
  - T 0.50-0.70 → upgrade non-adversarial → "suspicious" (HIGH risk)
  - T ≥ 0.70  → force "adversarial" (HIGH risk)
  - drift = cosine distance from clean reference embedding gallery

Windows-specific settings (currently active in config.py):
  - NUM_WORKERS = 4  (change to 0 if DataLoader hangs)
  - pin_memory = True (GPU optimization — revert to False if CPU-only)

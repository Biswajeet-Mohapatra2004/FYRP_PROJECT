1. PROJECT OVERVIEW
================================================================================

TITLE:
  Explainable, Multi-Agent AI-Based Adversarial Threat Detection &
  Decision Framework for Facial Biometric Systems

CORE INNOVATION (3 novelties):
  1. Dynamic Threshold Engine — replaces static/adaptive thresholds using
     4-factor context-aware scoring (confidence, anomaly, embedding drift,
     image quality). Addresses the "one-size-fits-all" limitation from
     Park et al. (2024) Risk Analysis paper.
  2. Advocate–Judge Multi-Agent Debate — LLM-backed Proponent & Opponent
     agents argue, a Judge agent validates. Implements LLM-as-a-Judge
     paradigm (prevents hallucination propagation from single-agent systems).
  3. Grad-CAM / XAI layer — shows WHERE on the face the adversarial
     perturbation was detected.

PIPELINE (end-to-end):
  Input Image
    → MTCNN Face Alignment & Cropping
    → Preprocessing (OpenCV, PIL) → Normalized tensor (1,3,224,224)
    → Feature Extraction (ResNet-18 CNN)
    → Dynamic Threshold Engine → T score, risk level
    → Conditional Debate Gate:
        T < 0.20  → fast-path LEGITIMATE (no LLM calls)
        T > 0.65  → fast-path ADVERSARIAL (no LLM calls)
        0.20–0.65 → full Advocate–Judge debate
    → Grad-CAM XAI heatmap
    → Final Decision Report (JSON + heatmap)

--- ADVERSARIAL ATTACK PIXEL-SPACE FIX (model/adversarial.py) ---
[✅] CRITICAL BUG FIXED in FGSM.perturb() and PGD.perturb().

     OLD (broken) behavior:
       - perturbation = epsilon * grad.sign()  [in normalized space]
       - adv = images + perturbation
       - adv = clamp(adv, 0, 1)  ← WRONG: normalized images span [-2.5, 2.5],
                                    not [0,1]. This clipped ~50% of pixels to 0,
                                    producing grey-blob artifacts. The clamp made
                                    epsilon changes have NO visible effect.

     NEW (correct) behavior:
       FGSM: denormalize → add epsilon*sign(grad) in pixel [0,1] space
             → clamp to [0,1] → re-normalize
       PGD:  all iterations work in pixel space; projection uses pixel ε-ball;
             re-normalize each step for model input; alpha auto-scales to ε/4

     RESULT: epsilon now has its standard meaning (max pixel change per channel
     in [0,1] space), matching published FGSM/PGD papers.

--- generate_test_attacks.py IMPROVEMENTS ---
[✅] Added --epsilon flag (applies to both FGSM and PGD).
     Added --fgsm_epsilon and --pgd_epsilon for independent control.
     PGD alpha auto-scales to epsilon/4 when epsilon is provided.
     Output filename encodes epsilon: tony_stark_fgsm_eps0.005.jpg
     Prints whether attack is imperceptible/subtle/visible.

6. THRESHOLD ENGINE REFERENCE (CURRENT VALUES)
================================================================================

Formula:
  T = 0.35*(1-confidence) + 0.30*anomaly + 0.20*drift + 0.15*(1-quality)

Decision boundaries (model/threshold.py):
  THRESHOLD_LOW  = 0.20  (T < this → trust CNN, fast-path)
  THRESHOLD_MID  = 0.40  (T in LOW–MID → upgrade normal→suspicious)
  THRESHOLD_HIGH = 0.65  (T in MID–HIGH → upgrade suspicious→adversarial;
                           T ≥ HIGH → always adversarial)

Debate gate (pipeline.py):
  DEBATE_LOWER = THRESHOLD_LOW  = 0.20  (fast-path LEGITIMATE)
  DEBATE_UPPER = THRESHOLD_HIGH = 0.65  (fast-path ADVERSARIAL)

Weights are now instance-configurable:
  DynamicThresholdEngine(w_confidence=0.35, w_anomaly=0.30,
                         w_drift=0.20, w_quality=0.15)

9. RESEARCH CONTEXT (FOR VIVA / PRESENTATION)
================================================================================

Your project addresses the gap found in:
  Park et al. (2024) "A Comprehensive Risk Analysis Method for Adversarial
  Attacks on Biometric Authentication Systems"
  → They analyzed 7 attack surfaces × 12 scenarios using FIXED thresholds
  → Fixed threshold = "one-size-fits-all" problem
  → Your solution: replace with dynamic threshold + agent-based reasoning

Key literature:
  1. Park et al. (2024)     — Risk analysis, fixed threshold gap
  2. Verheyen et al. (2023) — Adaptive thresholds still fail (doppelganger)
  3. Guo et al. (2025)      — Adversarial attacks survey (FGSM, PGD)
  4. Li et al. (2026)       — PhishDebate: multi-agent in cybersecurity
  5. Chen et al. (2026)     — G-DMAD: multi-agent debate improves reasoning
  6. Acharya et al. (2025)  — Agentic AI survey (justifies agent architecture)
  7. Apap et al. (2024)     — Explainable biometrics (justifies XAI)

3 Novelties to emphasize in viva:
  1. Dynamic threshold (replaces static/adaptive) — now with configurable weights
  2. Advocate-Judge agent debate (LLM-as-Judge, conditional gate for efficiency)
  3. Grad-CAM XAI integration (shows WHERE attack is on face)

Academic framing for epsilon (IMPORTANT for viva):
  Standard benchmark epsilon for L∞ adversarial attacks on ImageNet-scale
  tasks is ε=8/255 ≈ 0.031. Imperceptible range is ε≤0.01.
  Our system uses ε=0.03 (pixel space) as the primary evaluation point,
  which is the standard benchmark value in published FGSM/PGD literature.
  At ε=0.03, perturbations are borderline perceptible (~7.65/255 per pixel).

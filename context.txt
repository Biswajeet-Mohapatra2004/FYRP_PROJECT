1. PROJECT OVERVIEW
================================================================================

TITLE:
  Explainable, Multi-Agent AI-Based Adversarial Threat Detection &
  Decision Framework for Facial Biometric Systems

CORE INNOVATION (3 novelties):
  1. Dynamic Threshold Engine — replaces static/adaptive thresholds using
     4-factor context-aware scoring (confidence, anomaly, embedding drift,
     image quality). Addresses the "one-size-fits-all" limitation from
     Park et al. (2024) Risk Analysis paper.
  2. Advocate–Judge Multi-Agent Debate — LLM-backed Proponent & Opponent
     agents argue, a Judge agent validates. Implements LLM-as-a-Judge
     paradigm (prevents hallucination propagation from single-agent systems).
  3. Grad-CAM / XAI layer — shows WHERE on the face the adversarial
     perturbation was detected.

PIPELINE (end-to-end):
  Input Image
    → MTCNN Face Alignment & Cropping
    → Preprocessing (OpenCV, PIL) → Normalized tensor (1,3,224,224)
    → Feature Extraction (ResNet-18 CNN)
    → Dynamic Threshold Engine → T score, risk level
    → Conditional Debate Gate:
        T < 0.20  → fast-path (trust CNN prediction, no LLM calls)
        T > 0.65  → fast-path ADVERSARIAL (no LLM calls)
        0.20–0.65 → full Advocate–Judge debate
    → Grad-CAM XAI heatmap
    → Final Decision Report (JSON + heatmap)

--- EMBEDDING DRIFT FIX (model/threshold.py) ---

[✅] PROBLEM: Drift showed 0.0000 for every image even after gallery rebuild.
     ROOT CAUSE 1: _compute_drift() used max_sim (nearest-neighbour).
       After retraining to 100% accuracy, all face embeddings cluster on the
       unit sphere. Any face always finds one gallery entry with sim ≈ 0.99.
       max_sim ≈ 0.999 → drift = (1-0.999)/2 = 0.0005 ≈ 0.0000.
     ROOT CAUSE 2: The 512-d embedding head has NO contrastive loss.
       It is updated only through classifier backprop via shared backbone.
       After training, the MLP maps all faces to nearly identical unit vectors
       regardless of whether they are clean or adversarial.

     FIX: Blended probability-based drift (70%) + gallery cosine drift (30%).
       prob_drift    = 1.0 − P(normal)   [directly supervised, always reliable]
       gallery_drift = 1.0 − mean(top10_cosine_sim)  [OOD supplement]
       drift = 0.7 * prob_drift + 0.3 * gallery_drift

     Expected values after fix:
       Clean face:       P(normal)≈0.99 → drift ≈ 0.04
       FGSM ε=0.03:      P(normal)≈0.02 → drift ≈ 0.73
       PGD  ε=0.03:      P(normal)≈0.01 → drift ≈ 0.75

--- GRAD-CAM FGSM BLANK HEATMAP FIX (utils/visualize.py) ---

[✅] PROBLEM: FGSM images rendered a solid blue Grad-CAM overlay.
     ROOT CAUSE: FGSM creates uniform pixel-space sign-gradient noise.
       Layer4 (semantic) detects it via feature SUPPRESSION — negative
       gradient channels dominate. F.relu() zeroed all of them out.
       After relu: all zeros → normalized to all zeros → jet=solid blue.
     PGD was unaffected because structured iterative attacks produce strong
     positive spatial activations in specific face regions.

     FIX: When F.relu() discards >95% of the raw signal, fall back to
       abs(cam_raw) so the magnitude of suppressed activations is visible.
       PGD path is unchanged (positive activations survive relu normally).

5. THRESHOLD ENGINE REFERENCE (CURRENT VALUES)
================================================================================

Formula:
  T = 0.35*(1-confidence) + 0.30*anomaly + 0.20*drift + 0.15*(1-quality)

Drift computation (model/threshold.py):
  normal_prob = class_probabilities["normal"]
  prob_drift  = 1.0 - normal_prob                    (70% weight)
  gallery_drift = 1.0 - mean(top10_cosine_sim)       (30% weight)
  drift = 0.7 * prob_drift + 0.3 * gallery_drift

Decision boundaries:
  THRESHOLD_LOW  = 0.20  (T < this → fast-path, trust CNN)
  THRESHOLD_MID  = 0.40  (T in LOW–MID → upgrade normal→suspicious)
  THRESHOLD_HIGH = 0.65  (T in MID–HIGH → upgrade suspicious→adversarial;
                           T ≥ HIGH → always adversarial, fast-path)

Debate gate (pipeline.py):
  DEBATE_LOWER = 0.20  (fast-path: trust CNN)
  DEBATE_UPPER = 0.65  (fast-path: ADVERSARIAL)
  Between      → full multi-agent debate runs

Expected T values by epsilon (post-retraining):
  Clean face:     T ≈ 0.04–0.08   → LEGITIMATE (fast-path)
  ε=0.01 FGSM:   T ≈ 0.15–0.25   → LEGITIMATE or SUSPICIOUS boundary
  ε=0.02 FGSM:   T ≈ 0.25–0.40   → SUSPICIOUS (debate runs)
  ε=0.03 FGSM:   T ≈ 0.35–0.55   → SUSPICIOUS/ADVERSARIAL (debate runs)
  ε=0.03 PGD:    T ≈ 0.40–0.60   → SUSPICIOUS/ADVERSARIAL (debate runs)
  ε=0.05 PGD:    T ≈ 0.60–0.80   → ADVERSARIAL (may fast-path)

7. EPSILON DETECTION GUIDE
================================================================================

The model was trained on epsilon ∈ {0.01, 0.02, 0.03, 0.05} (recommended).
Single-epsilon training (only 0.03) creates a detection "floor" at ~0.02.

| Epsilon | Pixel change | Detection | Notes |
|---------|-------------|-----------|-------|
| < 0.01  | < 2.5/255   | Unreliable | Below floor; indistinguishable from sensor noise |
| 0.01    | 2.5/255     | Marginal   | Model just starts detecting at this range |
| 0.02    | 5.1/255     | Good       | Covered by multi-epsilon training |
| 0.03    | 7.6/255     | Strong     | Standard benchmark (ε=8/255 in papers) |
| 0.05    | 12.7/255    | Very strong| Clearly detectable; slight visual noise |
| > 0.05  | > 12.7/255  | Trivial    | Visibly distorted; always ADVERSARIAL fast-path |

For demo: Use ε=0.03. This is the sweet spot — imperceptible to human eye,
detectable by the model, and triggers the multi-agent debate (not fast-path).

8. KNOWN LIMITATIONS & DESIGN NOTES
================================================================================

1. EMBEDDING HEAD NOT CONTRASTIVELY TRAINED
   The 512-d embedding head has no triplet/contrastive loss — updated only
   through classifier backprop. After training, adversarial and clean face
   embeddings cluster together on the unit sphere. Pure gallery-based cosine
   drift is near-zero for everything. Fixed by blending with P(normal) drift.

2. FGSM HEATMAP INTERPRETATION
   FGSM uniform noise is detected via feature SUPPRESSION in layer4, not
   activation. Grad-CAM shows abs(cam) fallback for FGSM (broad/diffuse
   pattern). PGD shows focused face-region activation. Both are correct —
   they reflect fundamentally different attack structures.

3. DETECTION FLOOR AT ε < 0.01
   Perturbations below 0.01 pixel change are indistinguishable from normal
   image-to-image variation and JPEG quantization noise. This is a fundamental
   perceptual limit, not a code bug.

4. DEBATE TRIGGERS ONLY IN ε=0.02–0.05 RANGE
   T < 0.20 → fast-path (no LLM). T > 0.65 → fast-path ADVERSARIAL.
   The multi-agent debate (system's core novelty) only fires for ambiguous T.
   For demo, use ε=0.03 to ensure the debate runs.

11. RESEARCH CONTEXT (FOR VIVA / PRESENTATION)
================================================================================

Your project addresses the gap found in:
  Park et al. (2024) "A Comprehensive Risk Analysis Method for Adversarial
  Attacks on Biometric Authentication Systems"
  → They analyzed 7 attack surfaces × 12 scenarios using FIXED thresholds
  → Fixed threshold = "one-size-fits-all" problem
  → Your solution: replace with dynamic threshold + agent-based reasoning

Key literature:
  1. Park et al. (2024)     — Risk analysis, fixed threshold gap
  2. Verheyen et al. (2023) — Adaptive thresholds still fail (doppelganger)
  3. Guo et al. (2025)      — Adversarial attacks survey (FGSM, PGD)
  4. Li et al. (2026)       — PhishDebate: multi-agent in cybersecurity
  5. Chen et al. (2026)     — G-DMAD: multi-agent debate improves reasoning
  6. Acharya et al. (2025)  — Agentic AI survey (justifies agent architecture)
  7. Apap et al. (2024)     — Explainable biometrics (justifies Grad-CAM)

3 Novelties to emphasize in viva:
  1. Dynamic threshold (replaces static/adaptive) — configurable weights,
     4-factor score, context-aware per-image
  2. Advocate-Judge agent debate (LLM-as-Judge, conditional gate for
     efficiency — only fires for ambiguous cases)
  3. Grad-CAM XAI integration (shows WHERE attack is on face; FGSM=diffuse,
     PGD=focused — interpretable difference between attack types)

Academic framing for epsilon (IMPORTANT for viva):
  Standard benchmark epsilon for L∞ adversarial attacks on ImageNet-scale
  tasks is ε=8/255 ≈ 0.031. Imperceptible range is ε≤0.01.
  Our system uses ε=0.03 (pixel space) as the primary evaluation point,
  which is the standard benchmark value in published FGSM/PGD literature.
  At ε=0.03, perturbations are borderline perceptible (~7.65/255 per pixel).
